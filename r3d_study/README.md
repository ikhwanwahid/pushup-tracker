# R3D-18 3D CNN Study

Standalone module for comparing R3D-18 configurations on per-rep push-up form classification.

## Study Design

| Stage | Description |
|-------|-------------|
| **A** | Full-frame input, frozen backbone (FC only = 1,026 params) |
| **B** | YOLO-crop input, frozen backbone (FC only = 1,026 params) |
| **Unfreeze** | Best of A/B with 1 or 2 backbone blocks unfrozen |
| **HP Tune** | Grid search over LR and batch size on best config |

## Data Pipeline

No state machine. Rep boundaries are **manually annotated** by the team.

```
1. Record videos (2 reps each, good + bad)
2. Use annotate_helper.py to find start/end frame numbers
3. Fill in annotations.csv
4. Run notebook — it handles YOLO keypoint extraction automatically
```

## Files

| File | Purpose |
|------|---------|
| `model.py` | R3D-18 wrapper with `unfreeze_last_n_blocks()` |
| `datasets.py` | Full-frame and YOLO-crop video datasets |
| `training.py` | Train/evaluate loops and k-fold CV |
| `data_loader.py` | Reads annotations CSV, extracts YOLO keypoints |
| `annotate_helper.py` | Frame-by-frame video player for finding rep boundaries |
| `annotations_template.csv` | CSV template for team annotations |
| `notebook.ipynb` | Full study notebook |
| `test_r3d_study.py` | Unit tests |

## Annotation Format

```csv
video_filename,rep_number,start_frame,end_frame,label,shallow,partial_ext,hip_sag,hip_pike,uncontrolled,incomplete,annotator,notes
alice_bad_01.mp4,1,10,45,bad,1,0,0,0,0,0,alice,didn't go low enough
alice_bad_01.mp4,2,48,55,bad,0,0,0,0,0,1,alice,gave up halfway
```

Key columns: `video_filename`, `start_frame`, `end_frame`, `label` (good/bad).
Defect flags (shallow, partial_ext, etc.) are optional but useful for analysis.

## How to Annotate

```bash
python annotate_helper.py path/to/video.mp4
```

Controls: SPACE=next frame, LEFT=prev, F=+10, B=-10, Q=quit.
Note the frame numbers where each rep starts and ends.
